WALL Syntax Structure

[label:] [instruction] [# comment]

The above line is the syntactical structure for every line of code written in WALL.
Note: The brackets denote that the content is optional. Even though each portion is optional, the order is not.

  1. A line begins with an optional label, that is delimited by a colon.
    Note: Labels ARE case sensitive.
  2. The next part of the line is an Operation or Pseudo-Op,followed by its arguments.
    Note: Instructions and arguments are NOT case sensitive.
  3. Finally a line can end in an optional comment. (Anything that comes after the comment symbol "#" is treated as a comment, even labels and instructions.)

Reserved Characters and Words

These are characters and words that are used by WALL to Parse and interpret the code input into the Text Editor. Therefore using them as a label or a EQU label may cause unpredictable errors in the Assembler.

  1. Commas(,) - to deliminate arguments
  2. Quotations("") - used by DB to input strings into memory
  3. Apostrophes('') - used by DB to input strings into memory
  4. Mathematic Operators(= <) - used by JMP operations
  5. Register Names(R0-RF) - overloading these names can confuse the parser
  6. Operation Names - overloading these names can confuse the parser
  7. Pseudo-Op Names - overloading these names can confuse the parser

WALL has 16 Registers.

Registers are used as a form of "lag free" memory to store data temporarily as it is being manipulated.
Most of these are general purpose, and have no special uses, but the following have special purposes.

Register Purpose
RO Comparison Register - used for comparison in JMPLT and JMPEQ Op-Codes.
RD (RBP) Base Pointer Register - points to the base of the stack (Used for current Frame of Reference)
RE (RSP) Stack Pointer Register - points to the top value on the stack.
RF Display Register - takes the value placed into it and displays it's ASCII equivalent to the Display Console.

Operations

Operations are Instructions and their arguments. Each instruction has its own specific argument structure. All instructions compile into their Op-Code format and are stored into memory for execution. Note: All Instructions, except for RLOAD, are 2 Bytes long. RLOAD is 4 Bytes long, because it compiles as an immediate load, followed by itself.

  1. ADD
  2. AND
  3. CALL
  4. HALT
  5. ILOAD
  6. ISTORE
  7. JMP
  8. JMPEQ
  9. JMPLT
  10. LOAD
  1. MOVE
  2. OR
  3. POP
  4. PUSH
  5. RET
  6. RLOAD
  7. ROR
  8. RSTORE
  9. STORE
  10. XOR

Pseudo-Ops

Pseudo-Ops are similar to Operations, except that they do not compile as code directly into memory. Instead, they act as a kind of marker that tell the Assembler how to handle some kinds of associations and data.

  1. BSS
  2. DB
  3. EQU
  4. ORG
